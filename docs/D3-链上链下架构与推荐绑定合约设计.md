
# D3 链上/链下架构 + 推荐绑定合约设计（v1）

> 目标：在**不破坏现有链下账本系统**的前提下，把「唯一需要防篡改的关系数据（推荐绑定）」上链，
> 把「真金 USDT 流」继续锁在链上托管多签，把「利息/奖励计算」留在链下但做到**可验证、可自证、偿付透明**。
> 链：BNB Smart Chain（chainId `56`）　结算资产：USDT `0x55d398326f99059fF775485246999027B3197955`

## 1. 资金与代币模型（统一口径）

### 1.1 链下记账单位（都不是链上代币）

> **变更**：原「贿赂金 sD3」引擎**废除**；奖励系统收敛为**唯一的 UD3**。

| 单位 | 中文 | 含义 | 计价 | DB 列 | 由什么产生 |
|---|---|---|---|---|---|
| **D3** | — | 质押/生息计量单位、每日释放单位 | **1 D3 = 5 USDT**（管理价，可调） | `partner_stake_positions.staked_d3` / `partner_accounts.pending_d3_yield` | 质押头寸按 D3 价折算 |
| **UD3** | 反向金 / 推荐奖励 | **唯一奖励**：下线入金按上线档位 S1–S6（100%–150%）生成 | **1 UD3 = 1 USDT**（锚定，记账 USDT） | `partner_accounts.ud3_balance` | `partnerUd3Rules.ts` 引擎 |
| ~~sD3~~ | ~~贿赂金~~ | **已废除**，迁移/清算入 UD3 | — | ~~`sd3_balance`~~ | ~~`partnerSd3Rules.ts`~~（下线） |

**UD3 用途（严格限定，不可直接提取）：**
1. **转账给伞下下线**（`/partner/sd3-transfer` → 更名 UD3，校验对方在自己伞下）
2. **再质押**（`kind='ud3'`，2× 出局）

> **闭环设计**：UD3 永远**不能直接提现**。用户拿到真金 USDT 的**唯一路径**：
> `质押(UD3 或 USDT) → 每日释放 D3 → 闪兑 D3 → USDT(−3%)`。
> 好处：全网 UD3 不是即时兑付负债，真正的 USDT 兑付负债只来自「已释放但未闪兑的 D3」，偿付敞口可控。

### 1.2 质押来源与出局倍数

| 质押来源 | `kind` | 出局倍数 | 现状 | 说明 |
|---|---|---|---|---|
| **USDT 入金** | `partner_join` / `crowdfund_stake` | **6×** | ✅ 已有 (`exit_multiplier=6`) | 用户真金入金，锁 540 天 |
| **UD3 再质押** | `ud3`（新） | **2×** | ⚠️ 需改 | 因 1UD3=1USDT，本金=UD3 数量的 USDT 等值；扣 `ud3_balance`，不再产生新 UD3 业绩 |

> **已定**：sD3 废除，可再质押的唯一奖励是 **UD3**。落地时把再质押函数 `kind='sd3'`→`'ud3'`、扣 `sd3_balance`→`ud3_balance`；
> 历史 `sd3_balance` 余额一次性并入 `ud3_balance`（迁移脚本）。

### 1.3 D3 数量计价的利息模型（新口径 —— 与现状的关键差异）

**你要的模型：**
```
质押入场：  stakedD3   = depositUsdt / d3Price          （入场价折算，锁定 D3 本金数量）
每日释放：  dailyD3    = depositUsdt × 0.4% / d3Price(入场)  （每天释放的 D3 数量，固定）
出局判定：  按「释放 D3 的价值」计——每天释放的 D3 用当日 D3 价折算成 USDT 价值累计，
           累计价值 ≥ 本金 × 出局倍数(USDT 6× / UD3 2×) 即出局（最后一天按余额封顶）。
           ★ 无 540 天截断：头寸一直释放到价值出局；D3 涨价则更快到 6×。unlock_at 仅展示。
闪兑兑付：  usdtGross  = releasedD3 × d3Price(当日)
           fee        = usdtGross × 3%
           usdtNet    = usdtGross − fee                  （从闪兑钱包付给用户）
```

**现状代码（`partnerSettlement.ts`）是 USDT 计价：** `dailyYield = principal_usdt × 0.4%`，直接累进 `pending_usdt_yield`，闪兑就是提这笔 USDT，**没有 D3→USDT 这一步**。

**差异影响**：D3 涨价时，你的模型让用户闪兑拿更多 USDT；现状锁死 USDT。改这套需要动 §6 的 schema + 结算/闪兑逻辑。

## 3. 推荐绑定合约（核心新增）

> **已落地（`contracts/`，UUPS，20/20 测试通过）**：采用「**用户自付 gas**」方案——不用 relay / 不用 ERC-2771 转发器。以下 3.1/3.2 已按此更新；3.4 的 relay 流程作废（保留仅作历史对比）。

### 3.1 设计目标
- 绑定关系**上链、可公开审计**；用户自绑后**默认永久不可改**。
- 逻辑**可升级**（UUPS 代理，storage 在代理，升级不丢数据）。
- **用户自付 gas**：Privy 钱包直接调 `bind(upline)`（和现有转 USDT 存款同一套 viem+Privy 发交易路径），无 relay、无代付。
- 管理员**可纠错改绑**，但必须 **Turnkey 多签**（`REBIND_ADMIN_ROLE`）+ 链上 `Rebound(reason)` 事件，杜绝暗改。
- 合约层强制**单一上线 + 防环**（现有 Postgres 唯一约束挡不住，见 §8）。

### 3.2 合约清单（实际落地）

| 合约 | 角色 | 可升级 |
|---|---|---|
| `ReferralRegistry` (impl) | 业务逻辑：`bind` / `adminRebind` / 单一上线 + 防环 + 角色 | 是（UUPS） |
| `ERC1967Proxy` | 代理入口（delegatecall 到 impl，storage 在代理，数据永续） | — |

> UUPS 代理本身即「可升级逻辑 + 存储永续」，不再单拆 `ReferralStorage`；`D3Forwarder`（ERC-2771）已移除（用户自付 gas，不需要）。

### 3.3 逻辑合约接口（草案）

```solidity
interface IReferralRegistry {
    event Bound(address indexed user, address indexed upline, uint64 at, address relayer);
    event Rebound(address indexed user, address indexed oldUpline, address indexed newUpline, uint64 at, string reason);

    /// 用户自绑（经 Forwarder 转发，_msgSender() = 真实用户）。一次性、不可自改。
    function bind(address upline) external;

    /// 管理员纠错改绑：onlyRole(REBIND_ADMIN)，由 Turnkey 多签地址持有；强制留 reason 事件。
    function adminRebind(address user, address newUpline, string calldata reason) external;

    function uplineOf(address user) external view returns (address);
    function boundAt(address user) external view returns (uint64);
    function isBound(address user) external view returns (bool);
}
```

**校验规则（合约内强制）：**
- `upline != user`（禁自绑自己）
- `upline` 必须已 `isBound` 或为「创世根」白名单（保证连通到根）
- 防环：从 `upline` 向上回溯 N 层若遇到 `user` → revert（或用链下预检 + 链上浅检权衡 gas）
- `bind` 幂等：已绑再 `bind` → revert（单一上线）

### 3.4 绑定流程（用户自付 gas，已采用）

```
用户(Privy 钱包)
   │ 1. 前端直接调 ReferralRegistry.bind(upline)（viem+Privy 发交易，用户付 gas）
   ▼
链上 ReferralRegistry
   │ 2. 合约校验：单一上线、upline 已注册/为根、防环 → 写入 + Bound 事件
   ▼
后端监听 Bound / Rebound 事件 → 回写 Postgres referrals 表（索引缓存）
```

- 用户直接付 gas（几分钱 BNB），无 relay、无后端代付、无 EIP-712 签名环节。
- Postgres `referrals` 变成**可从链上事件重建的缓存**；奖励计算仍读它（快），但任何人可用链上事件核对，运营方无法暗改。
- 待建后端胶水：**事件监听同步**（`Bound`/`Rebound` → upsert `referrals`），以及前端在 `bind()` 成功后可选地 `report-tx` 让后端即时回写。

## 4. 计价层（为未来 ERC-20 / DEX 预留）

把当前散落的常量 `CROWDFUND_UNIT_PRICE_USDT = 5` 收敛成一个**单一价源接口**：

```ts
// _shared/d3Price.ts
export interface D3PriceSource { getD3PriceUsdt(): Promise<number>; }
// 现在：AdminConstPriceSource（读 DB 配置表 d3_price_settings，管理员可调）
// 未来：OraclePriceSource（读链上预言机/DEX TWAP）——发 ERC-20 上 DEX 时切换，无需改业务代码
```

所有折算（入场、日释放、闪兑）**只经此接口**，未来切预言机零改动。

## 5. 资金安全 + 防篡改机制

1. **真金 USDT**（已有）：存款 `verifyUsdtTransfer` 校验、兑付 Turnkey 多签签名、全程 `audit_log`。
2. **链下账本防篡改**（新增）：每日结算后把「全网 `{wallet → balances}`」算一棵 **Merkle 树**，把 **root 上链锚定**（`DailyStateAnchor` 合约，一天一笔、几分钱 gas）。用户可用 Merkle proof **自证某日余额**；运营方事后改库 → 对不上历史 root，篡改立即可查。
3. **偿付能力监控**（新增，最易爆的点）：闪兑钱包 USDT 储备必须 ≥ 全网 `pending` 兑付负债。
   - 现状固定 10% split（`SETTLEMENT_TO_FLASH_SWAP_PCT`）在提现高峰可能不足。
   - 加：`储备/负债` 比实时监控 + 低于阈值告警 + 从金库/清算动态补给。
4. **确定性重算**（已有）：每日 tick 幂等键 `partner_settlement_runs`，可复现。

## 6. Schema 变更清单（D3 数量计价）

> 新增 migration（建议 `027_d3_denominated_yield.sql`）：

```sql
-- 质押头寸：记录 D3 本金与 D3 出局上限（配合 §1.3）
alter table public.partner_stake_positions
  add column if not exists staked_d3        numeric(24,6),   -- = principal_usdt / d3Price(入场)
  add column if not exists d3_price_at_stake numeric(18,6),  -- 入场 D3 价快照
  add column if not exists daily_release_d3  numeric(24,6),  -- = principal_usdt × 0.4% / d3Price
  add column if not exists released_d3       numeric(24,6) not null default 0,  -- 累计已释放 D3
  add column if not exists exit_cap_d3       numeric(24,6);  -- = staked_d3 × exit_multiplier

-- 账户：待闪兑的 D3 数量（区别于现有 USDT 计价的 pending_usdt_yield）
alter table public.partner_accounts
  add column if not exists pending_d3_yield  numeric(24,6) not null default 0,
  add column if not exists lifetime_d3_yield numeric(24,6) not null default 0;

-- D3 价格配置（管理员可调；未来可被预言机覆盖）
create table if not exists public.d3_price_settings (
  id            smallint primary key default 1,
  price_usdt    numeric(18,6) not null default 5,
  source        text not null default 'admin_const',  -- 'admin_const' | 'oracle'
  updated_at    timestamptz not null default now()
);

-- 每日状态锚定（Merkle）
create table if not exists public.daily_state_anchors (
  settlement_date date primary key,
  merkle_root     text not null,
  tx_hash         text,
  leaf_count      integer not null default 0,
  created_at      timestamptz not null default now()
);
```

闪兑流程改动（`partnerYieldWithdraw.ts`）：入参从「USDT 金额」改为「D3 数量」，`usdtGross = d3 × d3Price`，再走现有 3% 费 + `yield_flash_withdraw` 出款。
