# 部署指南：链上合约 + 接线（BSC 主网 + 测试币）

> 网络：BSC 主网 chain 56。USDT 用**假币**（TestUSDT），现有 Turnkey 主网管道不变。
> 需要一个**部署钱包**（EOA + 少量真 BNB 付 gas）。合约在 `contracts/`（Foundry，已 26 测试通过）。

占位符：`<FAKE>`=TestUSDT 地址，`<REG>`=ReferralRegistry 代理地址，`<ANCHOR>`=DailyStateAnchor 地址，`<GAS>`=后端 gas 钱包地址。

---

## 0. 准备

```bash
cd contracts
export PRIVATE_KEY=0x你的部署钱包私钥        # 需有 BNB
export BSC_RPC_URL=https://bsc-dataseed.binance.org
export BSCSCAN_API_KEY=你的key              # 可选，用于 --verify（源码验证）
forge test                                  # 确认本地全绿（26 passed）
```

> `<GAS>` = 后端 gas 钱包地址：`echo $TURNKEY_GAS_WALLET_ADDRESS`，或调 `GET /treasury/health` 看 infrastructure。

## 2. 部署 ReferralRegistry（推荐绑定，UUPS 代理）

先用**部署 EOA 当 admin**（回迁完再转多签）。genesis root 默认 = admin。

```bash
export REFERRAL_ADMIN=$(cast wallet address --private-key "$PRIVATE_KEY")
forge script script/DeployReferral.s.sol --rpc-url "$BSC_RPC_URL" --broadcast --verify
```
记下 `ReferralRegistry(proxy):` 地址 = **`<REG>`**，并记下**部署区块号**（broadcast 日志里有，或 `cast block-number --rpc-url "$BSC_RPC_URL"`）= `<BLOCK>`。

## 3. 部署 DailyStateAnchor（每日锚定）

anchorer **必须是后端 gas 钱包**（它来发锚定交易）。

```bash
export ANCHORER=<GAS>
forge script script/DeployAnchor.s.sol --rpc-url "$BSC_RPC_URL" --broadcast --verify
```
记下 `DailyStateAnchor:` 地址 = **`<ANCHOR>`**。

## 8. 验证整条闭环

1. 打开 `/faketoken` → 连钱包 → **领取测试 USDT**（付几分钱真 BNB gas）。
2. 若开了链上绑定：绑定推荐人时会弹钱包**签 `bind()` 交易**（用户自付 gas）→ 后端核验回写。
3. 去 `/partner` 发起质押 → 拿到存款地址 → 把测试 USDT 打过去 → 后端入账 → 540 天头寸、每日释放 D3。
4. 闪兑释放的 D3 → USDT（闪兑钱包兑付，−3%）。
5. `GET /treasury/health` 看 `solvency`（储备/负债比）；`GET /partner/state-proof?date=YYYY-MM-DD` 拿自己余额的 Merkle 证明。

## 9.（硬化，推荐）把 admin 交给 Turnkey 多签

回迁完成后，将 `ReferralRegistry` 的三个角色（`DEFAULT_ADMIN_ROLE`/`REBIND_ADMIN_ROLE`/`UPGRADER_ROLE`）授予多签地址并从 EOA 撤销，`DailyStateAnchor` 的 owner 转给多签：

```bash
# 授予多签（示例，MS=多签地址）
cast send <REG> "grantRole(bytes32,address)" $(cast keccak "REBIND_ADMIN_ROLE") <MS> --private-key $PRIVATE_KEY --rpc-url "$BSC_RPC_URL"
# …对 DEFAULT_ADMIN_ROLE(0x00..00)、UPGRADER_ROLE 同理，最后 renounceRole 从 EOA 撤销
cast send <ANCHOR> "transferOwnership(address)" <MS> --private-key $PRIVATE_KEY --rpc-url "$BSC_RPC_URL"
```
> `DEFAULT_ADMIN_ROLE` 的 bytes32 是 `0x0000…0000`。撤销 EOA 前务必确认多签已能操作，避免锁死。

## 成本与注意

- Gas：3 次部署 + 回迁批量交易，约需几美元～几十美元的 BNB（视树规模）。
- **gas 钱包要有 BNB**：锚定交易、归集补 gas 都从它出。
- `SOLVENCY_MIN_RATIO=1.0` 表示储备需 ≥ 全网待闪兑负债；不足时 `/health` 报警，需给闪兑钱包补 USDT。
- 不设 `REFERRAL_REGISTRY_ADDRESS` / `DAILY_STATE_ANCHOR_ADDRESS` 时，系统自动退回「纯链下」模式，不受影响——可分步上线。
